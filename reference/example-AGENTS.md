# AGENTS.md

本文档用于约束本项目中的 AI / 自动化开发行为。开发时优先遵循本文件，其次遵循用户当前消息。

本文件与用户当前消息冲突时，先向用户指出冲突，再按用户当前消息执行。

## 基本原则

- 先读现有代码，再动手修改，优先沿用项目已有结构和写法。
- 不要臆测接口返回 —— 因为记忆会过期，排障前先 `curl` 真实接口或读真实日志。
- 写代码保持最少行数，能简单实现就不要引入复杂抽象。
- 不要为了兼容更多场景写大量分支 —— 因为未被当前需求验证的分支是净负担。
- 不要改无关文件，不要顺手重构 —— 因为它们会淹没本次改动的真实意图，让 review 无法聚焦。
- 不要回滚或覆盖工作区中已有的用户改动 —— 因为那是用户尚未提交的工作。
- 项目尚未上线，不需要兼容旧数据。表结构调整时直接按新设计修改。
- 不要写旧字段兼容分支或数据迁移兜底 —— 因为没有需要保护的生产数据，这些代码永远不会被执行。
- 每次写完代码不需要执行构建或测试，用户会自己做。

## 反复提醒沉淀

- 如果用户反复提醒同一个注意事项，先判定它是否跨项目通用。通用则回流框架素材库并明确告知；否则写入本文件对应章节。
- 补充时写成明确、可执行的规则。禁令写成「不要做 X —— 因为 Y」。
- 不要把踩坑记录当作规则沉淀 —— 因为它们是事实而非约束，应写入 `docs/pitfalls.md`。

## 外部知识查询

- 使用不熟悉的库前，按 Context7、专用 MCP、锁定 skill、WebFetch 的顺序查阅文档。
- 不要凭记忆写第三方库的 API —— 因为训练数据可能落后于当前版本。

## 后端规范

- 业务逻辑放在 `server/services/`，路由函数保持薄，只做参数校验、调用 service、返回。
- 启停钩子用 `@asynccontextmanager async def lifespan(app)`。
- 不要使用 `@app.on_event` —— 因为它已被 FastAPI 废弃。
- 持久化模型在 `server/models.py`，传输模型在 `server/schemas.py`，两者必须保持分离。
- 不要让一个类同时充当 ORM 实体和 API 响应模型 —— 因为持久化字段与对外字段的生命周期不同，混用会导致内部字段意外泄露。
- 不要用 f-string 构造 SQL —— 因为这是注入入口，一律参数化。

## 前端规范

- API 请求统一放在 `web/src/services/`，跨页面状态统一放在 `web/src/stores/`。
- 路由页面放在 `web/src/pages/`，页面私有组件放在各自页面目录的 `components/` 下。
- 不要为了单页面使用把组件放进 `web/src/components/` —— 因为共享目录会迅速堆满只有一个调用点的组件，让真正的复用组件难以辨认。
- 不要新增只做简单转发的组件，例如只 `return <X>{children}</X>` —— 因为它增加了一层无行为的间接，读者要多跳一次才能找到真实实现。
- 不要为了让组件「纯」而层层透传 props —— 因为透传链上每一层都会因无关变更而重新渲染。
- 组件私有样式用 Tailwind className。`web/src/styles/globals.css` 只放基础变量、全局重置和跨页面通用样式。
- 不要在全局样式文件里堆页面私有样式 —— 因为删除页面时没人敢删对应的样式，它会永久残留。
- 颜色一律取自 `web/src/lib/theme.ts` 的 token。
- 不要硬编码具体颜色值 —— 因为它们不会随主题切换，会造成深浅色模式下的视觉断裂。

## 文档规范

- `AGENTS.md` 是唯一事实源。`CLAUDE.md` 全文仅一行 `@AGENTS.md`。
- 不要维护两份都含技术细节的约束文件 —— 因为它们必然分叉，而 agent 会捡到过期的那份。
- 后续待办写到 `docs/progress/todo.md`，已实现但需人工验证的写到 `docs/progress/pending-test.md`。
- 不要在用户确认测试通过前把事项写进 `docs/overview/features.md` —— 因为未经验证的功能会让功能说明整体失去可信度。
- 不要在文档里写过期日期 —— 因为没人会回头更新它们。

## 发版本流程

- 发版本时，先把 `CHANGELOG.md` 的 `Unreleased` 整理成新的版本记录，保留空的 `Unreleased` 标题。
- 按当前版本号提升一个版本，更新根目录 `VERSION`，提交后打对应的 tag。
- 不要在发版流程中执行编译或测试 —— 因为发版是打标动作，验证应在此之前完成。

## 项目注意事项

- 技术栈：后端 FastAPI + SQLAlchemy + PostgreSQL；前端 React + Vite + Zustand + Tailwind。
- 项目尚未上线，无生产数据。构建与测试验证由用户本人负责。
- 图片与大 JSON 存 IndexedDB，`localStorage` 只放登录 token。
- 不要在文档中把本地缓存写成「已支持云同步」 —— 因为该功能尚未实现。
