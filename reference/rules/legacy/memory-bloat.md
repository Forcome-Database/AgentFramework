## Applies When
- 目标项目的 `AGENTS.md` 或 `CLAUDE.md` 行数超过 300。下文称超行的那个文件为 `<超行文件>`；两个都超行则两个都处理，各自独立判定。

## Do Not Apply When
- `## 项目注意事项` 章节的行数超过 `<超行文件>` 全文的一半，用 `awk '/^## 项目注意事项/,/^## [^项]/' <超行文件> | wc -l` 与 `wc -l < <超行文件>` 相比判定。该章节按定义随项目事实增长，不是膨胀，搬走它等于把项目事实藏起来。

  阈值必须是一个具体数字，不能写「集中在」 —— 因为「集中」无法判定，而 `AGENTS.md` 明令「不要使用『合理地』『适当地』『根据情况』这类措辞 —— 因为它们无法被判定」。这条排除条件所在的块是唯一一个真会动手改文件的自动档块，含糊在这里的代价最大。
- `<超行文件>` 由本框架最近一次 `init-agents` 生成且未经人工追加，即 `git log --oneline -- <超行文件>` 只有一条提交。

  **判定必须绑在 `<超行文件>` 上，不要写死 `AGENTS.md`。**因为标准工作流是「先 `init-agents` 立宪法，再 `refactor-legacy` 清算历史」，跑完 `init-agents` 后 `AGENTS.md` 恰好只有一条提交 —— 若把这条排除写死在 `AGENTS.md` 上，它在标准工作流下**恒真**，整个块被排除，那份 682 行的 `CLAUDE.md` 永远不会被拆，重构模式退化成「只会补一个索引文件」（证据：最终审查在一个 304 行 `CLAUDE.md` + 刚生成的 `AGENTS.md` 的靶子上实测复现）。
- 该文件没有二级标题（`grep -c '^## ' <文件>` 为零），无从按主题拆分。这时超行是单一长文，拆它需要判断内容归属，转由人工处理。

## Output Target
GENERATION_ONLY

## Rule
- 不要让约束文件无限追加 —— 因为它只增不减，直到无法被有效加载（证据：`reference/anti-patterns.md` 第 2 条，某项目的 `CLAUDE.md` 十天内从 10KB 增至 146KB，51 条踩坑记录漂移成三种格式，顶部的「全部阶段完成」在此后上百条记录中从未更新）。
- 拆分按**章节**进行：把整个二级标题连同其正文搬到 `docs/` 下的同名文件，在约束文件中原位留一行指针。整章搬迁不需要理解内容，是机械动作。
- 不要在搬迁时改写任何一行原文 —— 因为原文中的具体数字与报错信息才是证据，改写会把证据洗掉。搬迁后必须逐字节比对（`diff`）。
- 不要按「编号条目」判定踩坑记录 —— 因为编号只是列表标记，不含任何语义（证据：一个真实老项目的 `CLAUDE.md` 里，`grep -nE "^(##+ )?[0-9]+[.、)] "` 命中 22 条编号条目，逐条打开看：18 条是该项目的铁律治理清单，4 条是 PR 流程步骤，**一条踩坑记录都没有**。机械执行「把编号条目迁进 `docs/pitfalls.md`」，会把这个项目的宪法搬进一个叫「历史事实档案」的文件里——规则还在，但不再治理任何东西）。
- 不要自动迁移踩坑记录 —— 因为「什么算踩坑记录」需要理解内容的含义，而自动档只做机械的、内容保全的动作。哪些条目该进 `docs/pitfalls.md`，在阶段 6 作为建议报告给用户，由人决定。

## Verification
- 命令：`test "$(wc -l < <超行文件>)" -le 300`。**必须查 `<超行文件>`，不要写死 `AGENTS.md`** —— 因为膨胀的通常是 `CLAUDE.md`（真实案例 682 行），此时查一个 4 行的 `AGENTS.md` 恒真，本块会「验证通过」并提交一次什么都没干的改动。
- 命令：搬迁后逐章 `diff` 比对，原文必须逐字节一致。
- 自查：`<超行文件>` 中每个被搬走的章节，是否都留下了指向 `docs/` 下对应文件的一行指针？

## Legacy Scan
- 命令：`wc -l AGENTS.md CLAUDE.md`，任一超过 300 即命中。
- 命令：`grep -nE '^## ' AGENTS.md CLAUDE.md`，其输出即可供拆分的章节清单。只拆 `<超行文件>` 的章节。
- 命令：`grep -nE "^(##+ )?[0-9]+[.、)] " AGENTS.md CLAUDE.md`，其输出是**疑似**踩坑记录，仅供阶段 6 报告，**不作为自动迁移的依据**。

## Remediation
- 可逆性：自动
- 作用域：AGENTS.md, CLAUDE.md, docs/
- 动作：对 `<超行文件>`，按二级标题把章节整体搬到 `docs/` 下的同名文件，原位留一行指针，搬完 `diff` 逐字节核对。搬到该文件行数降至 300 以下为止，优先搬最长的章节。不改写任何一行原文，不重新分类任何条目。编号条目一律不动，只在报告中列出。
