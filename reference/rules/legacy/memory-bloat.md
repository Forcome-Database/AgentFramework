## Applies When
- 目标项目的 `AGENTS.md` 或 `CLAUDE.md` 行数超过 300。

## Do Not Apply When
- 超出 300 行的内容集中在 `## 项目注意事项` 章节，它按定义随项目事实增长，不是膨胀。
- 该文件由本框架最近一次 `init-agents` 生成且未经人工追加，即 `git log --oneline -- AGENTS.md` 只有一条提交。
- 该文件没有二级标题（`grep -c '^## ' <文件>` 为零），无从按主题拆分。这时超行是单一长文，拆它需要判断内容归属，转由人工处理。

## Output Target
GENERATION_ONLY

## Rule
- 不要让约束文件无限追加 —— 因为它只增不减，直到无法被有效加载（证据：`reference/anti-patterns.md` 第 2 条，某项目的 `CLAUDE.md` 十天内从 10KB 增至 146KB，51 条踩坑记录漂移成三种格式，顶部的「全部阶段完成」在此后上百条记录中从未更新）。
- 拆分按**章节**进行：把整个二级标题连同其正文搬到 `docs/` 下的同名文件，在约束文件中原位留一行指针。整章搬迁不需要理解内容，是机械动作。
- 不要在搬迁时改写任何一行原文 —— 因为原文中的具体数字与报错信息才是证据，改写会把证据洗掉。搬迁后必须逐字节比对（`diff`）。
- 不要按「编号条目」判定踩坑记录 —— 因为编号只是列表标记，不含任何语义（证据：一个真实老项目的 `CLAUDE.md` 里，`grep -nE "^(##+ )?[0-9]+[.、)] "` 命中 22 条编号条目，逐条打开看：18 条是该项目的铁律治理清单，4 条是 PR 流程步骤，**一条踩坑记录都没有**。机械执行「把编号条目迁进 `docs/pitfalls.md`」，会把这个项目的宪法搬进一个叫「历史事实档案」的文件里——规则还在，但不再治理任何东西）。
- 不要自动迁移踩坑记录 —— 因为「什么算踩坑记录」需要理解内容的含义，而自动档只做机械的、内容保全的动作。哪些条目该进 `docs/pitfalls.md`，在阶段 6 作为建议报告给用户，由人决定。

## Verification
- 命令：`test "$(wc -l < AGENTS.md)" -le 300`。
- 命令：搬迁后逐章 `diff` 比对，原文必须逐字节一致。
- 自查：约束文件中每个被搬走的章节，是否都留下了指向 `docs/` 下对应文件的一行指针？

## Legacy Scan
- 命令：`wc -l AGENTS.md CLAUDE.md`，任一超过 300 即命中。
- 命令：`grep -nE '^## ' AGENTS.md CLAUDE.md`，其输出即可供拆分的章节清单。
- 命令：`grep -nE "^(##+ )?[0-9]+[.、)] " AGENTS.md CLAUDE.md`，其输出是**疑似**踩坑记录，仅供阶段 6 报告，**不作为自动迁移的依据**。

## Remediation
- 可逆性：自动
- 作用域：AGENTS.md, CLAUDE.md, docs/
- 动作：按二级标题把章节整体搬到 `docs/` 下的同名文件，原位留一行指针，搬完 `diff` 逐字节核对。搬到行数降至 300 以下为止，优先搬最长的章节。不改写任何一行原文，不重新分类任何条目。编号条目一律不动，只在报告中列出。
